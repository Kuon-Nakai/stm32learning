CMSIS-Driver USART in SmartCard mode

STM32CubeMX Setup

The CMSIS-Driver USART requires:

  • Setup of USART/UART peripheral in SmartCard mode

Valid settings for various evaluation boards are listed in the table below:

Peripheral Resource NUCLEO-G474RE
USART Peripheral USART1
USART Mode SmartCard
TX Pin PC4
CK Pin PA8

For different boards, refer to the hardware schematics to reflect correct setup values.

The STM32CubeMX configuration for NUCLEO-G474RE with steps for Pinout, Clock, and System Configuration are listed below. Enter the values that are marked bold.

Pinout view

  1. Configure USART1 pins
    • PC4: USART1_TX
    • PA8: USART1_CK

Pinout & Configuration tab

  1. Under Connectivity open USART1:
    • Section Mode
      • Mode: SmartCard with Card Clock
    • Section Configuration
      • Parameter Settings: not used
      • User Constants: not used
      • NVIC Settings: configured later
      • DMA Settings: configured with DMA
      • GPIO Settings: check used pins
  2. Under System Core open GPIO:
    • Group By Peripherals - USART:
      Pin Name Signal on Pin GPIO mode GPIO Pull-up/Pull.. Maximum out.. User Label
      PC4 USART1_TX Alternate F.. No pull-up and no.. Very High .
      PA8 USART1_CK Alternate F.. No pull-up and no.. Very High .
  3. Under System Core open DMA:
    • Section Configuration
      • DMA1, DMA2
        • Click on Add button and add requests as in table below:
          DMA Request Channel Direction Priority
          USART1_RX DMA1 Channel 1 Peripheral to Memory Low
          USART1_TX DMA1 Channel 2 Memory to Peripheral Low
  4. Under System Core open NVIC:
    • Section Configuration
      • NVIC
        NVIC Interrupt Table Enabled Preemption Priority Sub Priority
        DMA1 channel1 global interrupt ON 0 0
        DMA1 channel2 global interrupt ON 0 0
        USART1 global interrupt ON 0 0

Clock Configuration tab

  1. Configure USART1 Clock.

Project Manager tab

  1. Open Advanced Settings:
    • enable Do Not Generate Function Call for MX_USART1_SMARTCARD_Init.
Note
The CMSIS Driver:USART (API):USART component in the Manage Run-Time Environment dialog adds multiple C source files to the project. The interface selection in STM32CubeMX selects the actual implementation that is compiled: IrDA, SmartCard, UART, or USART.
